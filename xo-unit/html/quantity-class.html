Quantity

#include <xo/unit/quantity.hpp>

The primary data structure for interacting with xo-unit is the template class xo::unit::quantity.

template<typename Unit, typename Repr = double>
class quantity

represents a scalar quantity; enforces dimensional consistency at compile time.

  • Unit is a type identifying dimension and scale attaching to this quantity. Unit must satisfy unit_concept<Unit>

  • Repr is a type used to represent quantity values, scaled by Unit. Repr must satisfy numeric_concept<Repr>

A quantity’s run-time state consists of exactly one Repr instance: sizeof(quantity<Unit, Repr>) == sizeof(Repr)

Type Traits

using unit_type = Unit

type capturing the units (and dimension) of this quantity

using repr_type = Repr

type used for representation of this quantity

template<dim BasisDim>
using find_bpu_t = unit_find_bpu_t<unit_type, BasisDim>

report this quantity’s basis-power-unit type for a given basis dimension

Example: 

auto q = 1.0 / (qty::milliseconds(5) * qty::seconds(100.0));
q.unit_cstr(); // "ms^-2"

using tmp = q.find_bpu_t<dim::time>;

tmp::c_native_dim;          // dim::time
tmp::c_native_unit;         // native_unit_id::second
tmp::scalefactor_type::num; // 1
tmp::scalefactor_type::den; // 1000
tmp::power_type::num;       // -2
tmp::pwoer_type::den;       // 1

template<dim BasisDim>
using basis_scale_type = typename find_bpu_t<BasisDim>::scalefactor_type::type

report this quantity’s scalefactor type for given basis dimension

static inline constexpr char const *unit_cstr()

abbreviation for this quantity’s units

This string literal is constructed at compile-time by concatenating abbreviations for each basis-power-unit. For implementation see:

  • xo::unit::native_unit_abbrev_helper (in xo/unit/basis_unit.hpp) for each native dimension

  • xo::unit::scaled_native_unit_abbrev (in xo/unit/basis_unit.hpp) last-resort handling for scaled native dimensions

  • xo::unit::scaled_native_unit_abbrev (in xo/unit/unit.hpp) specializations for scaled native dimensions

Constants

template<dim BasisDim, typename PowerRepr = int>
static constexpr PowerRepr basis_power = from_ratio<PowerRepr, typename find_bpu_t<BasisDim>::power_type>()

report exponent of BasisDim in dimension of this quantity

For example: 

auto q = qty::milliseconds(5) * qty::seconds(1);
int p1 = q.basis_power<dim::time>; // p1 == 2
int p2 = q.basis_power<dim::mass>; // p2 == 0

Conversion Methods

Amount-preserving conversion to quantities with different units and/or representation.

template<typename Unit2, typename Repr2 = repr_type>
inline constexpr quantity<Unit2, Repr2> with_unit() const

convert to quantity representing the same amount, but changing units and perhaps representation.

These two expressions are equivalent: 

q.with_unit<units::millisecond>();
quantity<units::millisecond, q::repr_type>(q);

template<typename BasisUnit2, typename Repr2 = repr_type>
inline constexpr auto with_basis_unit() const

produce quantity scaled according to BasisUnit2, representing the same value as *this.

For example:

auto q1 = 1.0 / minutes(1) * kilograms(2.5);         // q1 = 2.5kg.min^-1
auto q2 = q1.with_basis_unit<units::millisecond>();  // q2 in kg.ms^-1

Motivation is ability to chain rescaling to reach desired compound unit

auto q3 = q1.with_basis_unit<units::second>()
            .with_basis_unit<units::gram>();   // q3 in g.s^-1

template<typename Quantity2>

multiply this quantity x by another quantity y.

Result will propagate dimension and units appropriately. If x and y use conflicting scale factors for a dimension, adopt scalefactor from x.
