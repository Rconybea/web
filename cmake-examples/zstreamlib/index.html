Zstream Library

template<typename CharT, typename Traits = std::char_traits<CharT>>
class basic_zstream : public std::basic_iostream<CharT, std::char_traits<CharT>>

iostream implementation with automatic compression/uncompression

Example 1 - create a .gzip file 

zstream zs(0, "path/to/foo.gz", ios::out);

zs << "Some text to be compressed" << endl;
zs.close();

Example 2 - read from a .gzip file 

zstream zs(0, "path/to/foo.gz", ios::in);

while (!zs.eof()) {
    string x;
    zs >> x;

    cout << "input: [" << x << "]" << endl;
}

Example 3 - attach existing streambuf 

ios::openmode mode = ...;

zstream zs(0, nullptr, mode);

unique_ptr<streambuf> sbuf = ...;  // streambuf for compressed data
int fd = -1;                       // or actual file descriptor,  if known

zs.rdbuf()->adopt_native_sbuf(std::move(sbuf), fd);

Template Parameters:

  • CharT – character type for stream elements

  • Traits – character traits; typically std::char_traits<CharT>

inline std::streamsize read_until(char_type *s, std::streamsize n, bool check_delim_flag, char_type delim)

read characters up to delimiter

Read up to n-1 chars, into memory s[0]..s[n-2] Stop when either:

  • s[] is full (contains n-1 chars)

  • check_delim_flag is true, and reached character delim

Similar to .get(s,n,delim) except that:

  • read_until returns the number of characters read instead of *this

  • read_until includes writes delim (if encountered) to s

read_until ignores the noskipws flag

Note

We want read_until() as building block for overload that does not require caller to supply buffer size.

Return values:

number – of chars written to s[], excluding trailing null.

Parameters:

  • s – write characters to this array.

  • n – available space in s. Will not write past s[n-1]

  • check_delim_flag – true to stop after first occurrence of delim

  • delim – if check_delim_flag is true stop after first occurrence of delim in input

Post:

if read_until() returns nr with nr>0, then s[nr-1] is null (i.e. char_type())

inline std::string read_until(bool check_delim_flag, char_type delim, uint32_t block_z = 4095)

read characters up to delimiter and package into a string

Note

implementation is O(n) for return value of length n.

Parameters:

  • check_delim_flag – If true read until after first occurrence of delim. Otherwise read until end of input.

  • delim – Stop after first occurrence of this character. Ignored if check_delim_flag is false.

  • block_z – Use temporary strings of this size to assemble result.

Return values:

template<typename CharT, typename Traits = std::char_traits<CharT>>
class basic_zstreambuf : public std::basic_streambuf<CharT, std::char_traits<CharT>>

Implementation of std::basic_streambuf that provides automatic streaming inflation/deflation to gzip format.

Example - allocating buffer space from constructor. 

zstreambuf zbuf;

std::unique_ptr<xfilebuf> p(new filebuf());
if (p->open("path/to/file.gz", std::ios::in))
    zbuf.adopt_native_sbuf(std::move(p), -1)

Example - allocating buffer space after constructor returns. 

zstreambuf zbuf(0);

zbuf.alloc(64UL*1024UL);

std::unique_ptr<xfilebuf> p(new filebuf());
if (p->open("path/to/file.gz", std::ios::in))
    zbuf.adopt_native_sbuf(std::move(p), -1)

Note

Only uses calling thread; not threadsafe.

Template Parameters:

  • CharT. – Typename for (uncompressed) characters. Compressed stream always comprises uint8_t's

  • Traits. – Typename for streambuf traits object. Typically this will be std::char_traits<CharT>.

inline basic_zstreambuf(size_type buf_z = 64 * 1024, native_handle_type fd = -1, std::unique_ptr<std::streambuf> native_sbuf = std::unique_ptr<std::streambuf>(), std::ios::openmode mode = std::ios::in)

Creates new zstreambuf.

Note

Taking care with alignment: if CharT is a wide character type, probably want/need aligned buffers for uncompressed data.

Parameters:

  • buf_z – Buffer size for inflation/deflation algorithm. Actual buffer memory consumption is 4x this value, to account for:

    • .in_zs_.z_in_buf_ buffer compressed input

    • .in_zs_.uc_out_buf_ buffer uncompressed input

    • .out_zs_.uc_in_buf_ buffer uncompressed output

    • .out_zs_.z_out_buf_ buffer compressed output Can use 0 to defer buffer allocation

  • fd – Native handle (file descriptor), if known, that native_sbuf is using. Providing to anticipate c++26, and for the sake of python IOBase.fileno() when using this stream from python.

  • native_sbuf – streambuf for doing compressed i/o. If this is a filebuf, it must be in an open state

  • mode – Openmode bitmask. combination of std::ios::in, std::ios::out, std::ios::binary. Other openmode bits are ignored

template<typename CharT, typename Traits = std::char_traits<CharT>>
class basic_xfilebuf : public std::basic_filebuf<CharT, std::char_traits<CharT>>

Minimal extension of gcc’s std::basic_filebuf, for the sole purpose of exposing file descriptor.

We want to support the iostream::native_handle() method, in advance of c++26.

Note

  1. For file i/o, we want to know the file descriptor associated with a stream:

    • To permit using epoll(), io_uring_enter() or similar mechanism to schedule i/o activity on linux

    • To implement python IOBase.fileno() when using the stack: 

      zstream.py / pyzstream / ::zstream / ::zstreambuf
(python)     (pybind11)     (c++)        (c++)

  2. The GNU 12.3.0 stdc++ library has (under include/c++/12.3.0): 

    [x86_64-unknown-linux-gnu/bits/c++io.h]        typedef FILE __c_file
[x86_64-unknown-linux-gnu/bits/basic_file.h]   __c_file * __basic_file<char>::_M_cfile (private)
                                               int __basic_file<char>::fd()            (public)
[include/fstream]                              __basic_file<char> basic_filebuf<..>::_M_file (protected)

  3. We can’t get this from a std::filebuf via public API (at least, prior to c++26 filebuf::native_handle()):

  4. There’s also no public API path to attach a separately-created file descriptor with a filebuf.

  5. This leaves us the narrowest path to squeeze through, followed here:

    • Write class inheriting std::filebuf

    • In this class, provide wrapper for std::filebuf::open()

    • Wrapper can access file descriptor (after successful .open()) using self->_M_file.fd()

    • Record file descriptor separately (after that can use only the unextended std::filebuf api
